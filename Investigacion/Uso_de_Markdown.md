# Documentación sobre Markdown y su Uso en Proyectos de Software

##  1. ¿Qué es Markdown?

Markdown es un lenguaje de marcado ligero diseñado para crear texto con formato mediante una sintaxis simple y legible.  
Su propósito principal es permitir que los documentos sean fáciles de escribir en texto plano y también convertibles a HTML u otros formatos.

###  ¿Por qué se utiliza en proyectos de software?

- Permite documentar proyectos de forma clara y rápida.  
- Es ampliamente compatible con plataformas como **GitHub**, **GitLab** y **Bitbucket**.  
- Se usa para crear archivos clave como `README.md`, `CONTRIBUTING.md`, `CHANGELOG.md`, entre otros.  
- Su sintaxis es sencilla, lo que reduce tiempo de trabajo.  
- Es portable y soportado por diversas herramientas de documentación.

##  2. Ejemplos prácticos de Markdown

###  Encabezados

```markdown
# Título 1
## Título 2
### Título 3
```

###  Lista con viñetas

```markdown
- Elemento A
- Elemento B
- Elemento C
```

### Lista numerada

```markdown
1. Paso uno
2. Paso dos
3. Paso tres
```

### Tablas

```markdown
| Nombre | Rol | País |
|--------|------|-------|
| Ana    | Desarrolladora | Ecuador |
| Luis   | QA | Perú |
| Sara   | PM | México |
```

### Enlaces
 
 ```markdown
 [Ir a GitHub](https://github.com)
```

## 3. Ventajas de usar Markdown en GitHub
- Visualización automática: GitHub renderiza los archivos .md sin necesidad de conversiones.

- Documentación organizada: Permite crear documentación clara directamente dentro del repositorio.

- Colaboración eficiente: 
    - Soporta control de versiones.

    - Permite revisar cambios (diffs).

    - Facilita contribuciones de otros usuarios.

- Compatible con GitHub Pages: Ideal para sitios web de documentación generados desde Markdown.

- Ideal para wikis y manuales: GitHub ofrece wikis basadas completamente en Markdown.
